# GHS Markdown Editor — BL-05: Windows Taskbar Jump List

## Context
This is a backlog feature addition to the GHS Markdown Editor cross-platform app. All 10 phases are complete. BL-06 and BL-02 are also complete. This session implements BL-05 only.

Read `docs/SPEC.md` and `docs/NOTES.md` before starting.

---

## Goal
Show recent files in the Windows taskbar Jump List — the menu that
appears when the user right-clicks the app's taskbar button. Each
recent file appears as a clickable entry that opens directly in the app.

This is Windows-only. No changes needed for macOS.

---

## Step 1 — Add NuGet package

Add the Windows-only package for Jump List support:

```bash
dotnet add package System.Windows.Shell --version 4.0.1
```

Wait — `System.Windows.Shell` is WPF-only and won't work in Avalonia.

Instead, use the Windows Shell COM API directly via P/Invoke, or use
the `Microsoft.WindowsAPICodePack-Shell` package. Check availability:

```bash
dotnet add package WindowsAPICodePack-Shell
```

If that package is not available or causes conflicts, implement using
the raw Windows `ICustomDestinationList` COM interface via P/Invoke.
The P/Invoke approach is self-contained and has no NuGet dependency.

Use whichever approach builds cleanly. Document which was used in
the report.

---

## Step 2 — Create JumpListService

File: `Services/JumpListService.cs`

```csharp
public class JumpListService
{
    // Updates the Windows taskbar Jump List with the given recent files.
    // No-op on non-Windows platforms.
    public void UpdateJumpList(IEnumerable<string> recentFilePaths);
}
```

Implementation notes:

- Guard with `if (!OperatingSystem.IsWindows()) return;` at the top
  of `UpdateJumpList()` so it compiles and runs safely on macOS.
- Use `[SupportedOSPlatform("windows")]` attribute on the method.
- Each recent file entry:
  - Title: `Path.GetFileName(filePath)`
  - Arguments: `filePath` (passed as the command-line argument to the app)
  - Icon: the app executable itself (index 0)
  - Category: "Recent Files"
- Limit to the most recent 10 files (Jump List has limited space).
- If the COM approach is used, implement via `ICustomDestinationList`:
  ```
  CoCreateInstance(CLSID_DestinationList) →
  BeginList() →
  AppendCategory("Recent Files", items) →
  CommitList()
  ```
- If `WindowsAPICodePack-Shell` is available, use:
  ```csharp
  JumpList jumpList = JumpList.CreateJumpListForCurrentApplication();
  jumpList.AddUserTasks(...)
  jumpList.Refresh();
  ```

Register as singleton in DI.

## Step 3 — Wire to FileService.RecentFilesChanged

In `ViewModels/MainWindowViewModel.cs` (or `App.axaml.cs`), after
DI setup, subscribe to `FileService.RecentFilesChanged`:

```csharp
_fileService.RecentFilesChanged += (_, _) =>
{
    var recent = _fileService.GetRecentFiles().Take(10);
    _jumpListService.UpdateJumpList(recent);
};
```

Also call `UpdateJumpList` once on startup with the current recent
files list so the Jump List is populated immediately on launch.

## Key Technical Notes

**Platform guard:** The entire JumpListService implementation must
be guarded with `OperatingSystem.IsWindows()` checks or
`[SupportedOSPlatform("windows")]` attributes. Without this,
the macOS build will fail or crash.

**App executable path:** The Jump List entries need the full path
to the app executable for the icon and launch path. Use:
```csharp
var exePath = Environment.ProcessPath ?? 
              Process.GetCurrentProcess().MainModule?.FileName ?? "";
```

**COM registration:** The Jump List is tied to the app's Application
User Model ID (AppUserModelID). For an unsigned app, Windows uses
the executable path. For a properly installed app (via Inno Setup),
the AppUserModelID can be set explicitly. For v1.0, the default
(exe path based) is sufficient.

**DI registration:** Register `JumpListService` as a singleton in
`App.axaml.cs` alongside other services.
