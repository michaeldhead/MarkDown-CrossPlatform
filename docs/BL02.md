# GHS Markdown Editor — BL-02: Typewriter + Focus Mode

## Context
This is a backlog feature addition to the GHS Markdown Editor cross-platform app. All 10 phases are complete. BL-06 is also complete. This session implements BL-02 only.

Read `docs/SPEC.md` and `docs/NOTES.md` before starting.

---

## Goal
Add a Focus Mode that combines two writing aids:
- **Typewriter scrolling** — the active line stays vertically centered in the editor as you type.
- **Prose dimming** — text outside the active paragraph fades to 20% opacity, keeping attention on the current block.

Focus Mode is a toggle — off by default, on when activated.

---

## Step 1 — Add FocusMode state to MainWindowViewModel

In `ViewModels/MainWindowViewModel.cs`:

```csharp
public bool IsFocusModeActive { get; private set; }

public IRelayCommand ToggleFocusModeCommand { get; }
```

Wire in constructor:
```csharp
ToggleFocusModeCommand = new RelayCommand(() =>
{
    IsFocusModeActive = !IsFocusModeActive;
    OnPropertyChanged(nameof(IsFocusModeActive));
    _focusModeAction?.Invoke(IsFocusModeActive);
});
```

Add field and setter:
```csharp
private Action<bool>? _focusModeAction;
public void SetFocusModeAction(Action<bool> action) => _focusModeAction = action;
```

Persist `IsFocusModeActive` to `SettingsService` so it survives restarts.
Add `bool FocusMode { get; init; } = false;` to `AppSettings`.

## Step 2 — Typewriter scrolling in the editor

In `Views/MainWindow.axaml.cs`, wire focus mode behavior on the
TextEditor when `IsFocusModeActive` is true:

**Typewriter scroll:** After every caret position change, when
Focus Mode is active, scroll the editor so the active line is
vertically centered:

```csharp
_editor.TextArea.Caret.PositionChanged += (_, _) =>
{
    if (!_mainVm.IsFocusModeActive) return;

    // Get the visual position of the caret line
    var caretLine = _editor.TextArea.Caret.Line;
    var editorHeight = _editor.Bounds.Height;

    // Scroll so caret line is at center
    var lineHeight = _editor.TextArea.TextView.DefaultLineHeight;
    var targetOffset = _editor.Document.GetLineByNumber(caretLine).Offset;
    var visualPos = _editor.TextArea.TextView.GetVisualPosition(
        new TextViewPosition(caretLine, 1),
        VisualYPosition.LineMiddle);

    var currentScroll = _editor.TextArea.TextView.ScrollOffset.Y;
    var targetScroll = currentScroll + visualPos.Y - (editorHeight / 2.0);

    _editor.ScrollToVerticalOffset(Math.Max(0, targetScroll));
};
```

## Step 3 — Prose dimming in the preview WebView

When Focus Mode is active, inject CSS into the preview WebView
that dims all block elements except the one matching the current
active block (the element with the `.ghs-active` class):

In `PreviewViewModel.cs` or `MainWindow.axaml.cs`, add a method
`ApplyFocusModeStyles(bool active)`:

```csharp
public async Task ApplyFocusModeStyles(bool active)
{
    if (active)
    {
        await _webView.InvokeScript(@"
            (function() {
                var style = document.getElementById('ghs-focus-style');
                if (!style) {
                    style = document.createElement('style');
                    style.id = 'ghs-focus-style';
                    document.head.appendChild(style);
                }
                style.textContent =
                    'body > *:not(.ghs-active) { opacity: 0.2; transition: opacity 0.3s ease; }' +
                    'body > .ghs-active { opacity: 1.0; transition: opacity 0.3s ease; }';
            })();
        ");
    }
    else
    {
        await _webView.InvokeScript(@"
            (function() {
                var style = document.getElementById('ghs-focus-style');
                if (style) style.remove();
            })();
        ");
    }
}
```

Call `ApplyFocusModeStyles()`:
- When Focus Mode is toggled on/off.
- After every `NavigateToString()` call (re-navigation removes
  injected styles — re-inject if Focus Mode is active).
  In the `NavigationCompleted` handler, after injecting JS
  listeners, call `ApplyFocusModeStyles(IsFocusModeActive)`.

## Step 4 — Focus Mode toolbar button

In `Views/MainWindow.axaml`, add a Focus Mode toggle button to
the toolbar:

```xml
<ToggleButton Content="Focus"
              IsChecked="{Binding IsFocusModeActive, Mode=OneWay}"
              Command="{Binding ToggleFocusModeCommand}"
              ToolTip.Tip="Toggle Focus Mode (Ctrl+Shift+F)"
              Margin="4,0,0,0" />
```

Style when active: `--accent` (#4A9EFF) background, white text.
Style when inactive: standard toolbar button appearance.

## Step 7 — Restore Focus Mode on startup

In `MainWindowViewModel` constructor, after loading settings:
```csharp
if (_settings.FocusMode)
{
    IsFocusModeActive = true;
    // _focusModeAction not yet wired at this point — it will
    // be applied when MainWindow calls SetFocusModeAction().
    // MainWindow should check IsFocusModeActive after wiring
    // and apply if true.
}
```

In `MainWindow.axaml.cs`, after calling `SetFocusModeAction()`:
```csharp
if (_mainVm.IsFocusModeActive)
    _mainVm.SetFocusModeAction?.Invoke(true); // apply on startup
```
